Detailed Description

The hardware library allows telling the stack that a certain hardware peripheral is in use by the application, so that the stack does not use the peripheral at the same time. Also vice versa, hardware library tells application if it is trying to use hardware peripheral used by the stack. The stack needs to know remaining battery voltage or chip temperature to do periodic RF calibrations on some platforms, and that may require the stack to access hardware peripherals that the application also uses.

The application may use a peripheral either intermittently or permanently. If the stack is unable to measure battery voltage periodically, because the application has reserved that particular peripheral permanently, the application must use the lib_state->setEnergy() function in the State library to tell the stack how much energy is left.

This library also allows the activation of a hardware peripheral. In fact, some peripheral may be needed by the application or the stack with a concurrent access as the activation of an external crystal for more precision.

typedef app_res_e(* app_lib_hardware_is_peripheral_activated_f)(app_lib_hardware_activable_peripheral_e peripheral, bool *activated_p, uint32_t timeout_us)

Request for peripheral status.

The given timeout_us is the maximum time to wait for peripheral activation. Each peripheral has a maximum timeout dependent on the platform.

Parameters
peripheralHardware peripheral to check status
activated_pPointer to store the status: true if activated, false otherwise
timeout_usMaximum time to wait for activation
Returns
If peripheral is not recognized or not reservable, APP_RES_RESOURCE_UNAVAILABLE is returned. If peripheral was not previously activated, APP_RES_INVALID_CONFIGURATION. If timeout_us is too long APP_RES_INVALID_VALUE is returned. Otherwise, APP_RES_OK is returned and activated_p is updated.
Note
Each peripheral has a maximum timeout dependent on the platform

typedef uint16_t(* app_lib_hardware_read_supply_voltage_f)(void)

Ask the last read supply voltage value from the stack. The returned value is in millivolts.

The stack periodically measures the supply voltage for its own use. The measurement is done about once every minute. If the application has reserved the peripheral that is used to do supply voltage measurements, the returned value is invalid.

Returns
Last sampled supply voltage, LSB is millivolts
Note
The stack normally samples the supply voltage once every minute, but reserving the ADC periheral for long periods of time may cause the sampling to happen less often, or not at all

typedef int32_t(* app_lib_hardware_read_temperature_f)(void)

Ask the last read chip temperature from the stack.

On some radio architectures, the stack needs to periodically measure chip temperature to do RF calibrations. The measurement is done about once every minute. On architectures where RF calibrations are not done, the returned value is invalid, usually 0. If the application has reserved the peripheral that is used to do chip temperature measurements, the returned value is also invalid.

Returns
Last sampled temperature, LSB is 0.01 degrees C

typedef app_res_e(* app_lib_hardware_reserve_peripheral_f)(app_lib_hardware_peripheral_e peripheral)

Reserve a hardware peripheral for application use. When a peripheral is in application use, the stack will not touch it. The peripheral parameter, app_lib_hardware_peripheral_e is the hardware peripheral to reserve.

If peripheral is not recognized or the stack is currently using the peripheral, APP_RES_RESOURCE_UNAVAILABLE is returned. Otherwise, APP_RES_OK is returned

Parameters
peripheralHardware peripheral to reserve
Returns
Result code, APP_RES_OK if successful, APP_RES_RESOURCE_UNAVAILABLE if the peripheral is currently in stack use or is not possible to reserve on this platform
